Certificate Desk - Project Documentation

Introduction

Certificate Desk is a comprehensive web application designed to automate the process of generating, managing, and verifying certificates. The system provides an intuitive interface for organizations to create professional certificates with embedded QR codes for verification purposes.

The application addresses the common challenges faced by educational institutions and organizations in certificate management, including manual generation, verification authenticity, and bulk processing requirements.

Abstract

This project implements a full-stack certificate management system that combines modern web technologies with efficient backend processing. The system features template-based certificate generation, participant management, QR code integration for verification, and comprehensive export capabilities.

Key innovations include visual positioning tools for template customization, mobile-responsive verification pages, and dual storage architecture supporting both local and cloud-based file management. The system is designed to scale from small organizations to large institutions with thousands of certificates.

Objectives

  • Automate certificate generation process to reduce manual effort
  • Provide secure certificate verification through QR codes
  • Enable bulk processing for large-scale certificate distribution
  • Create intuitive template management with visual positioning tools
  • Implement mobile-responsive verification interface

System Architecture

The Certificate Desk follows a three-tier architecture:

Presentation Layer

  • Responsive web interface
  • Mobile-optimized verification pages
  • Interactive template positioning tools

Business Logic Layer

  • Certificate generation engine
  • QR code creation and management
  • Template processing and positioning
  • Export and reporting functionality

Data Layer

  • PostgreSQL database for structured data
  • Vercel Blob storage for files
  • Local file system for development

Key Components

  • Template Management System
  • Participant Database
  • Certificate Generation Engine
  • Verification System
  • Delivery Log Tracking
  • Export Module

Database Schema

Templates Table:
- id (Primary Key)
- name, stored_path
- text positioning (x_ratio, y_ratio, font_size, align, color)
- QR positioning (x_ratio, y_ratio, size)
- archival status and timestamps

Participants Table:
- id (Primary Key)
- name, email
- creation and update timestamps

Certificates Table:
- id (Primary Key)
- participant_id, template_id (Foreign Keys)
- certificate details (number, year, full_id)
- file paths (pdf_path, qr_code_path)
- status and delivery tracking
- verification URL

Certificate Delivery Log Table:
- id (Primary Key)
- certificate_id (Foreign Key)
- delivery_method (email, download, manual)
- delivery_status (sent, failed, pending)
- delivery_timestamp
- error_message (if failed)
- recipient_email
- retry_count
- created_at, updated_at

API Endpoints

  • GET /api/templates - Retrieve all templates
  • POST /api/templates - Upload new template
  • GET /api/participants - Get participants list
  • POST /api/participants/import - Import CSV participants
  • POST /api/certificates/generate - Generate certificates
  • GET /api/certificates/verify/:id - Verify certificate
  • GET /api/certificates/:id/download - Download PDF
  • GET /api/certificates/export/excel - Export to Excel
  • GET /api/certificates/delivery-log - Get delivery log
  • POST /api/certificates/:id/delivery - Update delivery status

Security Features

  • Input validation on all endpoints
  • File type restrictions for uploads
  • Parameterized database queries
  • Secure file storage with Vercel Blob
